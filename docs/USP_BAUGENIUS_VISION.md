# BAUGENIUS: Das System das DEIN Unternehmen wird

> **"Nicht du lernst das System. Das System lernt dich."**

---

## Der fundamentale Unterschied

### Alte Welt (Jede Handwerker-Software heute)

```
Softwarehersteller definiert "Best Practice"
            ↓
Berater implementiert Standard-Prozess
            ↓
Handwerker MUSS sich anpassen
            ↓
"So macht man das halt"
            ↓
Frustration, Workarounds, Excel nebenbei
```

**Das Problem:** Der Softwarehersteller in München weiß nicht, wie Deine Baulöwen in Hamburg arbeitet. Trotzdem soll Dennis arbeiten wie SAP/Craftboxx/OpenHandwerk es vorsieht.

### Neue Welt (BauGenius)

```
Handwerker arbeitet wie ER will
            ↓
System beobachtet still
            ↓
System erkennt Muster
            ↓
System lernt den Betrieb
            ↓
System WIRD der Betrieb
```

**Der Unterschied:** Das System formt sich um DICH. Nicht umgekehrt.

---

## Die Vision: Unternehmens-DNA digitalisiert

Jeder Betrieb hat eine einzigartige DNA:

- **Wie** er Projekte startet
- **Was** er vor der Baustelle vorbereitet  
- **Welche** Materialien er bevorzugt
- **Wie schnell** seine Leute sind
- **Wie** er mit Kunden kommuniziert
- **Wann** er bestellt
- **Wo** er einkauft
- **Was** bei ihm schiefgehen kann

Diese DNA existiert heute nur in Köpfen. Wenn der Chef krank ist, steht alles. Wenn ein Mitarbeiter geht, geht Wissen verloren.

**BauGenius extrahiert diese DNA und macht sie zum System.**

**Alte Welt:**
```

→ Du trägst manuell ein

```

**BauGenius:**

## Technische Umsetzung

### Kernprinzip: Observe → Pattern → Suggest → Confirm → Learn

```
1. OBSERVE:  System beobachtet jede Aktion
2. PATTERN:  System erkennt Wiederholungen
3. SUGGEST:  System schlägt vor (nach 3x gleich)
4. CONFIRM:  User bestätigt oder korrigiert
5. LEARN:    System passt sich an
```

### Datenstruktur für Lernfähigkeit

```sql
-- Jede Entscheidung wird getrackt
CREATE TABLE decision_log (
  id UUID PRIMARY KEY,
  decision_type TEXT,        -- 'material', 'planning', 'vorstep', etc.
  context JSONB,             -- Projekttyp, Kunde, Raum, etc.
  chosen_option TEXT,        -- Was wurde gewählt
  alternatives JSONB,        -- Was wurde NICHT gewählt
  user_id UUID,
  created_at TIMESTAMPTZ
);

-- Patterns werden extrahiert
CREATE TABLE learned_patterns (
  id UUID PRIMARY KEY,
  pattern_type TEXT,
  context_match JSONB,       -- Wann gilt dieses Pattern
  suggested_action JSONB,    -- Was vorschlagen
  confidence DECIMAL,        -- Wie sicher (0-1)
  use_count INTEGER,         -- Wie oft bestätigt
  last_confirmed_at TIMESTAMPTZ
);
```

### Beispiel: Material-Learning (existiert bereits)

```sql

CREATE TRIGGER trg_learn_product
```
